# TPACK 수업 설계 전문 컨설턴트

대한민국 선생님들을 위한 TPACK(Technological Pedagogical Content Knowledge) 수업 설계 도우미 웹 애플리케이션입니다.

## 🎯 주요 기능

- **수업 주제 및 대상 입력**: 학년, 과목, 주제를 직접 입력
- **교수법(P) 선택**: 강의, 탐구, 토론, 협력, 게임 등 다양한 수업 방식 선택
- **기술 도구(T) 선택**: 1인 1기기, 모둠별 기기, 교사만 기기 등 환경 선택
- **AI 기반 수업 설계안 생성**: OpenAI GPT-4를 활용한 전문적인 수업 설계안 자동 생성

## 📋 사전 요구사항

- Node.js (v14 이상)
- npm 또는 yarn
- OpenAI API 키 (프로그램 실행 후 화면에서 입력)

## 🚀 설치 및 실행 방법

### 로컬에서 실행하기

#### Windows PowerShell 사용 시:

```powershell
# 1. 프로젝트 폴더로 이동
cd "C:\Users\user\코딩\TPACK 수업레시피"

# 2. 의존성 패키지 설치 (처음 한 번만)
npm install

# 3. 서버 실행
npm start
```

**⚠️ 중요:** `코딩\TPACK 수업레시피`를 직접 입력하지 마세요! `cd` 명령어를 사용해야 합니다.

#### 서버 실행 확인:

서버가 정상적으로 실행되면 다음과 같은 메시지가 표시됩니다:
```
🚀 TPACK 수업 설계 도우미 서버가 포트 3000에서 실행 중입니다.
📖 브라우저에서 http://localhost:3000 로 접속하세요.
```

#### 브라우저에서 접속:

서버가 실행되면 브라우저에서 다음 주소로 접속하세요:

```
http://localhost:3000
```

**⚠️ 주의:** HTML 파일을 직접 열지 마세요! 반드시 서버를 실행한 후 `http://localhost:3000`으로 접속해야 합니다.

#### 개발 모드 (자동 재시작):

코드를 수정할 때마다 자동으로 서버가 재시작됩니다:

```bash
npm run dev
```

### 5. API 키 입력

프로그램을 처음 실행하면 **API 키 입력 화면**이 표시됩니다.

**Google AI Studio API 키 발급 방법:**
1. [Google AI Studio](https://aistudio.google.com/apikey)에 접속
2. Google 계정으로 로그인
3. "Create API Key" 버튼 클릭하여 새 API 키 생성
4. 생성된 키를 프로그램 화면의 입력창에 입력

**중요:**
- API 키는 브라우저의 localStorage에 안전하게 저장됩니다
- 한 번 입력하면 다음 접속 시 자동으로 사용됩니다
- API 키를 변경하려면 화면 우측 상단의 "🔑 API 키 변경" 버튼을 클릭하세요
- API 키는 서버로 전송되어 Google Gemini API 호출에만 사용되며, 서버에 저장되지 않습니다

## 📖 사용 방법

1. **수업 주제 입력**: 예) "초5 과학 - 태양계", "고2 문학 - 관동별곡"
2. **수업 방식 선택**: 원하는 교수법을 복수 선택 (체크박스)
3. **기술 도구 선택**: 사용 가능한 기술 환경을 복수 선택
4. **수업 설계안 생성** 버튼 클릭
5. 생성된 수업 설계안 확인 및 활용

## 🛠️ 기술 스택

- **백엔드**: Node.js + Express
- **프론트엔드**: HTML, CSS, JavaScript (Vanilla)
- **AI**: Google Gemini API (Google AI Studio)
- **기타**: dotenv (환경 변수 관리), cors (CORS 처리)

## ⚠️ 주의사항

- OpenAI API 사용 시 비용이 발생할 수 있습니다. API 사용량을 확인하세요.
- API 키는 브라우저의 localStorage에 저장되므로, 브라우저 데이터를 삭제하면 다시 입력해야 합니다.
- 프로덕션 환경에서는 HTTPS를 사용하는 것을 권장합니다.

## 🔧 문제 해결

### 서버가 시작되지 않는 경우
- Node.js 버전 확인: `node --version` (v14 이상 필요)
- 포트 3000이 이미 사용 중인 경우 `.env`에서 다른 포트로 변경

### API 오류가 발생하는 경우
- 프로그램 화면에서 API 키가 올바르게 입력되었는지 확인
- API 키에 충분한 크레딧이 있는지 확인
- 인터넷 연결 상태 확인
- 브라우저 개발자 도구(F12)의 콘솔에서 오류 메시지 확인
